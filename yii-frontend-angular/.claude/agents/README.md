# Catálogo de agentes

## Mapa task → agente

| Tarefa | Agente |
|---|---|
| Implementar controller, service, view, componente, filtro, diretiva | `frontend-developer` |
| Integrar com endpoint, revisar contrato de API, ajustar CORS | `backend-integration` |
| Decisão arquitetural, planejamento de feature, mudança em ambos os repos | `tech-lead` |
| Revisar PR, checar qualidade, validar padrões | `code-reviewer` |
| Levantar casos de teste, escrever bug report, validar edge cases | `qa-engineer` |
| Investigar bug, rastrear causa raiz | `debugger` |
| Otimizar watchers, reduzir chamadas de API redundantes | `performance-engineer` |
| Decisões de UI, Bootstrap, acessibilidade, feedback visual | `ui-ux-assistant` |
| Documentar endpoint, criar plano, escrever ADR | `documentation-writer` |
| Configurar ambiente, resolver conflito de porta, CI | `devops` |
| Transferir contexto entre sessões ou agentes | `handoffs` |

## Regra de context budget

Por sessão: `context-pack.md` + **1 standard** + **1 skill ou plano**. Não carregar tudo de uma vez.
